Clients and Servers

Computers connected to the Web are called clients and servers.

  • Clients are the typical Web user's Internet-connected devices (for example, your computer connected to your Wi-Fi; or your phone conneced to your mobile network) and Web-accessing software available on those devices (useually a webbrowser like Firefox or Chrome).
  • Servers are computer that store webpages, sites, or apps. When a client device wants to access a webpage, a copy of the webpage is downloaded from the server onto the client machine to be displayed in the user's web browser.

So What Acutally Happens?

  1. The browser goes to the DNS server and finds the real address of the server that the website lives on.
  2. The browser send an HTTP requst message to the server asking it to send a copy of the website to the client.
  3. If the server approves the client's request, the server sends the client on a "200 OK" message, which means "Permission granted! Here you go!", and then starts sending the website's files to the browser in a series of small chunks.
  4. The broswer assembles the small chunks into a complete website and displays it to you!
